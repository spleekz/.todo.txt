# .todo.txt

Язык для описания списка дел / идей.

## Тип файла

Язык предназначен для использования в файлах с расширением `.todo.txt`.

## Язык

Основная задача языка - кратко и понятно описать список дел; активные и выполненные задачи.

*При описании синтаксиса языка для знака переноса строки будет использоваться `\n`, а для знака табуляции - `\t`.*

### Активная задача

Для описания активной задачи используется следующая конструкция:

```
-\t[ Заголовок задачи ]\n
```

которая выглядит так:

```
-	[ Заголовок задачи ]

```

Маркером активной задачи является знак минус `-`.

Заголовок задачи отделяется от маркера знаком табуляции.

Определение задачи заканчивается переводом строки.

### Многострочная активная задача
Для описания активной задачи в несколько строк используется следующая конструкция:

```
-\tЗадача\n
\tПродолжение задачи\n
\tПродолжение продолжения задачи\n
```

Которая визуально выглядит так:

```
-	Задача
	Продолжение задачи
	Продолжение продолжения задачи

```

Продолжение задачи пишется на новой строке, с одним знаком табуляции в начале.

Для отделения одной задачи от другой, используется **минимум** одна пустая строка.

Итак, список активных задач может выглядеть следующим образом:

```
-	Задача 1

-	Задача 2
	Продолжение задачи 2

-	Задача 3
	Продолжение задачи 3

```


### Выполненная задача
Описание выполненной задачи имеет следующий вид:

```
\t*\tЗадача 1\n
\t\tПродолжение задачи 1\n
```

Что визуально выглядит так:

```
	*	Задача 1
		Продолжение задачи

```

Как видите, описание выполненной задачи очень схоже с описанием активной, с той лишь разницей, что маркером выполненной задачи является символ звёздочки `*`, а в начале каждой новой строки добавляется один знак табуляции, из-за чего задача визуально смещается вправо.

Дополнительный таб в начале каждой строки служит для визуального отличия выполненной задачи от активной, а также для поддержки функции сворачивания текста, доступной во многих текстовых редакторах, которая позволит при желании скрыть выполненные задачи.

Итак, список выполненных задач может выглядеть следующим образом:

```
	*	Задача 1

	*	Задача 2
		Продолжение задачи 2

	*	Задача 3
		Продолжение задачи 3

```


### Общие правила для всех типов задач

- Каждая задача объявляется с новой строки

- Каждая задача должна начинаться со своего маркера

- Каждая задача должна заканчиваться переводом строки


### Структура файла

`.todo.txt` файл делится на 2 секции, идущие в соответствующем порядке:

- Активные задачи
- Выполненные задачи


#### Секция активных задач

Включает в себя однострочный или многострочный заголовок ( опционально ), и список активных задач. Заголовок и задачи могут быть разделены произвольным количеством пустых строк.


#### Секция выполненных задач

Для определения начала секции выполненных задач служит специальный набор символов:

```
- * * -
```

Это помогает визуально отделить активные задачи от выполненных, а также озаглавить эту секцию.

Примеры:

```
------- * * -------
```

```
- * СДЕЛАНО * -
```

```
------- * Реализовано * -------
```

Любой из этих вариантов - валидный способ определить секцию выполненных задач.

Далее описывается список выполненных задач, который может быть отделён произвольным количеством пустых строк.


### Все специальные символы

- `-` - маркер активной задачи

- `*` - маркер выполненной задачи

- `- * * -` - начало секции выполненных задач

## Упрощение работы

Когда приходит время "выполнить" задачу или наоборот, вернуть её из выполненных в активные, приходится совершать довольно много действий:

1. Полностью выделить все строки задачи

2. Вырезать задачу из секции

3. Вставить задачу в другую секцию

4. Изменить маркер задачи

5. Увеличить / сократить количество табов на каждой строке задачи

При всём при этом, нужно попытаться не допустить "склеивания" других задач, или наоборот появления между ними лишних отступов, чтобы не нарушались правила описания задач и наглядность списка.

Все вышеперечисленные задачи могут быть автоматизированы с помощью программы, расположенной в папке `./src`.
